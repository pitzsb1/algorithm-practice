# timsort

파이썬의 list.sort()와 sorted()는 Timsort(팀소트) 알고리즘을 사용한다. Timsort는 리스트 안에서 이미 정렬되어 있는 작은 구간(run)을 먼저 찾아낸 뒤, 그 구간들을 합병 정렬 방식으로 합쳐서 전체를 정렬하는 알고리즘이다. 즉, 데이터 안에 존재하는 부분적 규칙성을 적극적으로 이용해 불필요한 비교를 줄이고, 필요할 때만 삽입 정렬을 적용해 작은 구간을 정리한다. 이런 방식 덕분에 실생활 데이터처럼 “어느 정도 정렬된 상태”에서는 매우 빠르게 동작한다. 이 알고리즘은 데이터 안에서 이미 정렬된 구간을 찾아 활용하기 때문에, 최선의 경우 O(n), 평균과 최악의 경우 O(n log n)의 시간 복잡도를 가진다. 또한 안정 정렬이기 때문에 같은 값을 가진 요소의 순서를 유지한다. 다른 알고리즘들과 비교하면 다음과 같다. 우선 버블 정렬(Bubble Sort)은 인접한 원소들을 반복적으로 비교하고 교환하는 방식으로, 최악과 평균 모두 O(n²)의 시간 복잡도를 가지며 매우 비효율적이다. 이에 비해 Timsort는 O(n log n) 수준이므로 버블 정렬보다 훨씬 빠르며 실제 데이터에서도 좋은 성능을 보인다. 또한 이분 탐색(Binary Search)은 정렬된 리스트에서 원소를 찾는 데 사용하는 알고리즘으로, 탐색 과정이 절반씩 줄어들기 때문에 시간 복잡도가 O(log n)이다. 이분 탐색 자체는 정렬 알고리즘이 아니지만, Timsort처럼 먼저 데이터를 정렬하는 데 O(n log n)이 필요하고, 그 이후 탐색 시에는 O(log n)의 빠른 속도를 낼 수 있다.
